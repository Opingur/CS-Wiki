
# 多态（Polymorphism）

## 一、什么是多态

多态（Polymorphism）：

```text
同一个接口
不同的实现
```

例如：

```cpp
Animal* a;

a = new Dog();
a->speak();

a = new Cat();
a->speak();
```

虽然调用的都是：

```cpp
a->speak();
```

但结果不同。

这就是多态。

---

## 二、为什么需要多态

假设有：

```cpp
class Dog {
public:
    void speak() {
        cout << "汪汪" << endl;
    }
};

class Cat {
public:
    void speak() {
        cout << "喵喵" << endl;
    }
};
```

如果没有多态：

```cpp
dogSpeak(dog);
catSpeak(cat);
```

需要写很多重复代码。

---

有了继承：

```cpp
class Animal {
public:
    virtual void speak() {
        cout << "Animal" << endl;
    }
};

class Dog : public Animal {
public:
    void speak() override {
        cout << "Dog" << endl;
    }
};

class Cat : public Animal {
public:
    void speak() override {
        cout << "Cat" << endl;
    }
};
```

就能统一处理。

---

## 三、虚函数（Virtual Function）

虚函数：

```cpp
virtual void speak();
```

作用：

```text
运行时决定调用哪个函数
```

而不是编译时决定。

---

### 没有virtual

```cpp
class Animal {
public:
    void speak() {
        cout << "Animal" << endl;
    }
};

class Dog : public Animal {
public:
    void speak() {
        cout << "Dog" << endl;
    }
};
```

```cpp
Animal* a = new Dog();

a->speak();
```

输出：

```text
Animal
```

因为编译器只看：

```cpp
Animal*
```

---

### 加上virtual

```cpp
class Animal {
public:
    virtual void speak() {
        cout << "Animal" << endl;
    }
};
```

```cpp
Animal* a = new Dog();

a->speak();
```

输出：

```text
Dog
```

因为运行时发现：

```cpp
a实际指向Dog对象
```

所以调用：

```cpp
Dog::speak()
```

---

## 四、override

```cpp
void speak() override
```

作用：

```text
告诉编译器：
我是重写父类函数
```

---

例如：

```cpp
class Dog : public Animal {
public:
    void speak() override {
        cout << "Dog" << endl;
    }
};
```

推荐永远写。

---

### 为什么？

假设手滑：

```cpp
void speek() override
```

编译器立即报错。

如果不写override：

```cpp
void speek()
```

编译通过。

但多态失效。

---

## 五、多态成立的三个条件

### 条件1

必须有继承

```cpp
Dog : public Animal
```

---

### 条件2

必须有虚函数

```cpp
virtual void speak()
```

---

### 条件3

必须通过父类指针或引用调用

```cpp
Animal* a = new Dog();

a->speak();
```

或者：

```cpp
Animal& a = dog;

a.speak();
```

---

缺一个都不是多态。

---

## 六、最经典的例子

```cpp
#include <iostream>
#include <vector>
using namespace std;

class Animal {
public:
    virtual void speak() {
        cout << "Animal" << endl;
    }

    virtual ~Animal() {}
};

class Dog : public Animal {
public:
    void speak() override {
        cout << "Dog" << endl;
    }

    ~Dog() override {}
};

class Cat : public Animal {
public:
    void speak() override {
        cout << "Cat" << endl;
    }

    ~Cat() override {}
};

int main() {
    vector<Animal*> zoo;

    zoo.push_back(new Dog());
    zoo.push_back(new Cat());
    zoo.push_back(new Dog());

    for (Animal* a : zoo) {
        a->speak();
    }

    for (Animal* a : zoo) {
        delete a;
    }

    return 0;
}
```

输出：

```text
Dog
Cat
Dog
```

---

这里循环根本不知道：

```text
到底是Dog还是Cat
```

只知道：

```cpp
Animal*
```

这就是多态。

---

## 七、纯虚函数

```cpp
virtual void speak() = 0;
```

称为：

```text
纯虚函数
```

---

例如：

```cpp
class Animal {
public:
    virtual void speak() = 0;
};
```

意思：

```text
Animal不知道怎么叫
必须由子类实现
```

---

## 八、抽象类

含有纯虚函数的类：

```cpp
class Animal {
public:
    virtual void speak() = 0;
};
```

称为：

```text
抽象类
```

特点：

```cpp
Animal a;
```

错误。

不能创建对象。

---

只能：

```cpp
Dog d;
```

---

## 九、析构函数为什么常写virtual

正确写法：

```cpp
virtual ~Animal() {}
```

原因：

```cpp
Animal* a = new Dog();

delete a;
```

保证：

```text
先析构Dog
再析构Animal
```

避免资源泄漏。

---

## 十、Java中的对应思想

C++：

```cpp
Animal* a = new Dog();
```

Java：

```java
List<Integer> list = new ArrayList<>();
```

本质一样：

```text
父类（接口）引用

指向

子类（实现类）对象
```

---

## 十一、一句话总结

多态成立条件：

```text
继承
+
虚函数
+
父类指针(引用)
```

最经典代码：

```cpp
Animal* a = new Dog();

a->speak();
```

多态本质：

```text
父类指针指向子类对象

运行时决定调用哪个函数
```
