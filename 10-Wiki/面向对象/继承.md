# 继承（Inheritance）

## 一、什么是继承

继承是一种代码复用机制。

子类（Derived Class）可以获得父类（Base Class）的成员变量和成员函数。

```cpp
class Animal {
public:
    void eat() {
        cout << "eating" << endl;
    }
};

class Dog : public Animal {
};
```

此时：

```cpp
Dog d;
d.eat();
```

合法。

因为：

```text
Dog is an Animal
```

狗是一种动物。

---

## 二、继承的作用

### 代码复用

不用重复写相同代码。

```cpp
class Animal {
public:
    string name;

    void eat() {
        cout << "eat" << endl;
    }
};

class Dog : public Animal {
};

class Cat : public Animal {
};
```

Dog 和 Cat 自动拥有：

```cpp
name
eat()
```

---

## 三、三种继承方式

假设：

```cpp
class Base {
public:
    int a;

protected:
    int b;

private:
    int c;
};
```

### 1.public继承

```cpp
class Derived : public Base {};
```

结果：

|Base|Derived|
|---|---|
|public a|public a|
|protected b|protected b|
|private c|不可访问|

特点：

```text
权限保持不变
```

最常用。

现实开发95%以上使用 public 继承。

---

### 2.protected继承

```cpp
class Derived : protected Base {};
```

结果：

|Base|Derived|
|---|---|
|public a|protected a|
|protected b|protected b|
|private c|不可访问|

特点：

```text
public变protected
```

很少使用。

---

### 3.private继承

```cpp
class Derived : private Base {};
```

结果：

|Base|Derived|
|---|---|
|public a|private a|
|protected b|private b|
|private c|不可访问|

特点：

```text
public和protected全部变private
```

极少使用。

---

## 四、访问权限

### public

任何地方都能访问

```cpp
obj.x;
```

---

### protected

只能类内部和子类访问

```cpp
class Base {
protected:
    int x;
};
```

---

### private

只能当前类访问

```cpp
class Base {
private:
    int x;
};
```

子类也访问不了。

---

## 五、using修改权限

可以重新调整继承成员的访问权限。

```cpp
class Base {
public:
    int x;
};

class Derived : private Base {
public:
    using Base::x;
};
```

结果：

```cpp
Derived d;
d.x = 100;
```

合法。

虽然是 private 继承，但 x 被重新开放为 public。

---

## 六、多继承

一个类可以继承多个父类。

```cpp
class A {};
class B {};

class C : public A, public B {};
```

此时：

```text
C同时拥有A和B的成员
```

---

## 七、菱形继承

经典问题：

```text
      A
     / \
    B   C
     \ /
      D
```

代码：

```cpp
class A {};

class B : public A {};

class C : public A {};

class D : public B, public C {};
```

问题：

```text
D中出现两份A
```

会产生：

```text
二义性（Ambiguous）
```

---

例如：

```cpp
d.vf();
```

编译器不知道：

```text
调用B那条路径的A
还是C那条路径的A
```

---

## 八、虚继承

用于解决菱形继承。

```cpp
class B : virtual public A {};

class C : virtual public A {};
```

此时：

```text
D中只保留一份A
```

结构：

```text
      A
     / \
    B   C
     \ /
      D
```

逻辑结构没变。

但内存中只有一个 A 对象。

---

## 九、构造顺序

创建子类对象时：

```cpp
Dog d;
```

执行顺序：

```text
父类构造
↓
子类构造
```

销毁顺序：

```text
子类析构
↓
父类析构
```

口诀：

```text
先构造父亲
再构造儿子

先销毁儿子
再销毁父亲
```

---

## 十、最重要的一句话

继承最核心的思想：

```text
is-a
```

例如：

```cpp
Dog : public Animal

Cat : public Animal

Student : public Person
```

表示：

```text
Dog是一种Animal

Cat是一种Animal

Student是一种Person
```

如果不满足 is-a 关系，

通常不应该使用继承。

## 十一、综合示例：把继承知识点跑一遍

```cpp
#include <iostream>
#include <string>
using namespace std;

class Animal {
public:
    string name;

    void eat() {
        cout << name << " is eating" << endl;
    }

protected:
    int age = 0;

    void setAge(int a) {
        age = a;
    }

private:
    int secret = 123;
};

class Dog : public Animal {
public:
    void init(string n, int a) {
        name = n;      // 访问父类public成员
        setAge(a);     // 访问父类protected函数
        // secret = 1; // 错：子类不能访问父类private成员
    }

    void show() {
        cout << "Dog name = " << name << endl;
        cout << "Dog age = " << age << endl;
    }
};

class Cat : public Animal {
public:
    void init(string n) {
        name = n;
    }
};

int main() {
    Dog d;
    d.init("Wangcai", 3);
    d.eat();
    d.show();

    Cat c;
    c.init("Mimi");
    c.eat();

    return 0;
}


```
```
```
Dog 继承 Animal
所以 Dog 拥有 Animal 的 public / protected 成员

外部只能访问 public
子类内部可以访问 public + protected
private 只有父类自己能访问

注意：`protected` 不是给外部用的，是给**子类内部**用的。  
所以：

```cpp
d.name = "Wangcai"; // 对，public
d.eat();            // 对，public
d.age = 3;          // 错，protected外部不能访问

```
```
```
|Base里的成员|public继承|protected继承|private继承|
|---|---|---|---|
|public|public|protected|private|
|protected|protected|protected|private|
|private|不可访问|不可访问|不可访问|
